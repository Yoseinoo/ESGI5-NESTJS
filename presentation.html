Relations entre entités

  • User ↔ Team — ManyToMany (table team_members)
  • Team → Project — OneToMany, CASCADE delete
  • Project → Task — OneToMany, CASCADE delete
  • Task → Comment — OneToMany, CASCADE delete
  • Task → User (assignee) — nullable, SET NULL on delete

Stratégie

  • LocalStrategy — valide email + password au login
  • JwtStrategy — vérifie le Bearer token sur toutes les routes
  • JwtAuthGuard appliqué globalement
  • @Public() pour exclure login et /health
  • RolesGuard + @Roles() pour le RBAC
  • Vérification de propriété dans UsersService.update()

Points de vigilance résolus

  • Race conditions
    Jest parallèle → conflits sur DB partagée → résolu par --runInBand
  • Enums PostgreSQL
    synchronize(true) ne supprime pas les types enum → TRUNCATE ... CASCADE
  • Mock findOne × 2
    create() appelle findOne deux fois → mockResolvedValueOnce

Difficultés & solutions

passwordHash exposé dans POST /users
TypeORM select: false ignoré par save() → appel de findOne(saved.id) après le save
JWT_EXPIRES_IN : erreur 500 au login
Valeur "1h" convertie via Number() → NaN → stocker en secondes : 86400
Enums PostgreSQL non supprimés entre les tests
synchronize(true) drop les tables mais pas les types → TRUNCATE ... CASCADE sans drop de schéma
Race conditions entre suites e2e
Jest lance les fichiers en parallèle sur la même DB → flag --runInBand dans le script npm
Prisma 7 — ruptures de syntaxe
Enums inline invalides, url supprimé du schéma → migration vers prisma.config.ts + enums multi-lignes

Ce que j'aurais amélioré

Compléter les services stubs
ProjectsService et CommentsService sont restés à l'état de squelettes CLI — implémentation TypeORM complète manquante
Pagination & filtres
GET /tasks retourne tout sans limite — pagination par offset ou curseur + filtres (statut, priorité)
Refresh tokens
Token à durée fixe sans renouvellement — ajout d'un refresh token révocable pour les sessions longues
Migration Prisma complète
La branche feature/prisma pose les bases mais TypeORM reste actif — migration totale pour profiter du typage fort et de omit
Tests e2e exhaustifs
Seuls auth et users sont couverts — tests pour teams, projects, tasks, comments et le flux WebSocket manquants
Rate limiting & sécurité
Pas de protection contre le brute-force au login — @nestjs/throttler + helmet
